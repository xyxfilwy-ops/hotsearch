## 编码规范

### 命名约定
- 文件: `kebab-case.ts`
- 组件: `PascalCase.tsx`
- 函数: `camelCase`
- 常量: `UPPER_SNAKE_CASE`

### 代码风格
- 使用 Prettier 格式化
- 使用 ESLint 检查
- 每个函数不超过 50 行
- 每个文件不超过 300 行

### Git Commit 规范
```
<type>(<scope>): <description>

类型：feat, fix, docs, style, refactor, test, chore
示例：feat(auth): 添加用户登录功能
```

## 重要约定 ⚠️

### 必须遵守
- 修改前先运行测试
- 敏感信息放 .env，不提交
- 新功能必须写测试

### 禁止操作
- ❌ 不要删除 .env 文件
- ❌ 不要直接修改 node_modules
- ❌ 不要在代码中硬编码密钥
